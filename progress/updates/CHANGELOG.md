# 版本更新日志

## [1.4.0] - 2026-03-09

### 🔧 修复问题

- **特征维度不匹配**：修复了特征工程模块和检测模型模块之间的特征维度不匹配问题
- **模型加载失败**：增强了模型加载逻辑，确保在模型文件不存在或加载失败时，系统能够自动切换到演示模式
- **导入语句错误**：修复了所有模块的导入语句，确保正确导入其他模块
- **异常处理不足**：增强了系统的异常处理能力，确保系统在各种异常情况下都能正常运行

### ✨ 功能改进

- **演示模式增强**：优化了演示模式的规则权重，提高钓鱼邮件识别准确率
- **系统稳定性**：优化了模型加载逻辑，提高系统启动速度
- **特征工程**：添加了缺失的 `has_external_script` 特征

## [1.1.0] - 2026-02-26

### ✨ 新增功能

#### 邮件文件上传
- 支持 `.eml` 和 `.msg` 格式文件上传
- 拖拽上传功能
- 文件大小限制：50MB
- 支持 UTF-8/GBK编码自动识别

#### 邮件详情页展示
- 添加"原始邮件内容"按钮
- 折叠式手风琴 UI 设计
- 显示 6 个内容板块:
  1. 邮件头信息 (Headers)
  2. 纯文本正文 (Text Body)
  3. HTML 正文 (HTML Body)
  4. URL 列表
  5. 附件列表
  6. 原始完整邮件
- 一键复制功能
- XSS 安全防护

### 🔧 优化改进

#### 日志优化
- 静默处理网络超时错误
- 减少冗余日志输出
- 提升用户体验

#### whois 库兼容性
- 适配新版 whois 库 API (`whois.get()` vs `whois.whois()`)
- 增强异常处理能力

### 🐛 修复问题

| 问题 | 文件 | 说明 |
|------|------|------|
| `from_addr` 未定义 | parse_email.py | 改为正确的 `from_raw` |
| time 模块缺失 | email_traceback.py | 添加 `import time` |
| 数据库表结构不一致 | alerts.db | 删除旧库重新初始化 |
| Unicode 编码错误 | test_system.py | 改用 ASCII 标识 |
