# 系统更新摘要 - 版本 1.5.0

## 更新类型
- [x] 功能增强
- [x] 架构优化
- [x] 数据库结构变更

## 更新内容

### 1. 核心功能升级：二分类到三分类
- **实现三分类检测**：将原有的二分类（是/否）升级为三分类（正常、可疑、钓鱼）
- **定义三档位阈值**：
  - L1 (正常)：置信度 < 0.4
  - L2 (可疑)：0.4 ≤ 置信度 < 0.7
  - L3 (钓鱼)：置信度 ≥ 0.7
- **引入硬性熔断机制**：满足特定条件直接判定为钓鱼邮件，跳过后续模型计算
  - 沙箱确认为恶意
  - SPF/DKIM/DMARC 全部失败且发件人域名与显示名称不匹配
  - URL 存在于已知黑名单中

### 2. 架构优化
- **分层决策漏斗**：
  1. 硬性规则熔断 (Kill Switch)
  2. 沙箱与高权特征检查
  3. 模型预测
  4. 溯源模块辅助加权
  5. 阈值切分
- **动态权重调整**：根据沙箱分析结果和溯源信息动态调整置信度
- **增强模型加载逻辑**：当模型加载失败时，自动降级到演示模式

### 3. 特征工程优化
- **新增特征**：
  - `attachment_risk_score`：附件风险评分 (0-10)，区分不同类型附件的风险等级
- **细化特征**：将 `has_attachment` 从布尔值升级为连续的风险评分
- **特征维度**：从 44 维扩展到 45 维

### 4. 数据库结构变更
- **替换字段**：将 `is_phish` 字段替换为 `label` 字段，支持三分类标签
- **更新索引**：将 `idx_is_phish` 索引替换为 `idx_label` 索引
- **统计逻辑**：更新所有统计接口，支持三分类统计

### 5. API 接口更新
- **分析接口**：返回三分类结果（label、confidence、reason）
- **统计接口**：新增可疑邮件统计数据
- **告警接口**：支持按标签过滤告警记录

### 6. 前端适配
- **可视化展示**：建议使用颜色区分三分类结果
  - 🟢 绿色：正常 (Safe)
  - 🟡 黄色：可疑 (Suspicious)
  - 🔴 红色：钓鱼 (Phishing)
- **人机协同**：为可疑邮件提供人工审核入口

## 技术实现

### 核心文件变更
- `src/detector.py`：实现三分类分析逻辑，添加硬性熔断和动态权重调整
- `src/features.py`：添加 `attachment_risk_score` 特征，优化特征提取
- `src/app.py`：更新 API 接口和数据库操作，支持三分类
- `src/detector.py`：更新特征列顺序，添加新特征权重

### 决策流程
1. **硬性熔断检查**：检查沙箱结果、认证失败和黑名单URL
2. **沙箱高危检查**：处理高风险文件但沙箱未分析的情况
3. **模型预测**：使用 LightGBM 模型或演示模式进行预测
4. **融合决策**：综合模型概率和沙箱风险，计算最终置信度
5. **阈值切分**：根据置信度确定最终分类

## 预期效果
- **降低误报**：通过三分类机制，减少正常邮件被误判为钓鱼邮件的情况
- **人机协同**：可疑邮件交由人工审核，提高检测准确性
- **动态防御**：对于新型攻击，判定为可疑比直接放过更安全
- **可解释性**：提供详细的判定理由，增强系统透明度

## 后续建议
- **阈值调优**：根据实际业务场景调整三分类阈值
- **反馈闭环**：收集用户对可疑邮件的反馈，持续优化模型
- **可视化增强**：在前端展示更详细的风险分析结果
- **规则更新**：定期更新硬性熔断规则，适应新型钓鱼攻击
