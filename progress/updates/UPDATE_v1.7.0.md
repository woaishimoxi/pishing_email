## 更新内容

### 1. 项目问题分析与修复

#### 严重问题

##### 1.1 敏感凭证明文存储 (安全问题)
- **问题描述**：`config/api_config.json` 中 VirusTotal API Key 和邮箱密码以明文形式存储
- **影响**：严重安全漏洞，可能导致凭证泄露
- **建议修复**：将敏感凭证移至环境变量，删除配置文件中的明文密码

##### 1.2 数据库路径不一致
- **问题描述**：
  - `src/app.py` 使用 `data/alerts.db`
  - `src/utils/database.py` 使用 `data/phishing_detector.db`
- **影响**：两个模块使用不同的数据库文件，数据不互通
- **建议修复**：统一数据库路径

##### 1.3 数据库功能被禁用
- **问题描述**：
  - `src/app.py:178-179` 的 `init_db()` 调用被注释掉
  - `src/app.py:302-303` 数据库保存功能被注释
  - `src/app.py:317-318` 异步任务被注释
- **影响**：所有邮件分析结果不会持久化存储
- **建议修复**：重新启用数据库功能

#### 中等问题

##### 1.4 前端模板未正确使用
- **问题描述**：
  - `src/templates/` 目录存在 `dashboard.html` 和 `report.html`
  - Flask 配置 `static_folder='../static'`，模板文件未被正确渲染
- **影响**：前端页面可能无法正常加载
- **建议修复**：修改 Flask 配置以正确使用模板

##### 1.5 特征维度不一致
- **问题描述**：
  - README 声称 44 维特征
  - `detector.py` 定义了 39 维特征
- **影响**：特征向量与模型可能不匹配
- **建议修复**：统一特征维度定义

##### 1.6 Dockerfile 启动命令错误
- **问题描述**：`Dockerfile:41` 使用 `python -m src.app`
- **影响**：Docker 容器无法正常启动
- **建议修复**：改为 `python src/app.py`

##### 1.7 缺少预训练模型文件
- **问题描述**：`models/phish_detector.txt` 文件不存在
- **影响**：系统只能以演示模式运行
- **建议修复**：添加预训练模型文件或确保演示模式正常工作
